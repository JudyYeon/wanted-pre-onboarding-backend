# CQRS와 MongoDB 개념 및 차이점


## 1. MongoDB란?

- **MongoDB**는 NoSQL 계열의 문서 지향(Document-oriented) 데이터베이스입니다.
- 데이터를 JSON과 유사한 BSON(Binary JSON) 형태로 저장합니다.
- 스키마가 유연하고, 확장성이 좋아 빅데이터와 실시간 애플리케이션에 적합합니다.
- 주로 **데이터 저장소(DB)** 역할을 하며, CRUD(Create, Read, Update, Delete) 작업을 수행합니다.

---

## 2. CQRS란?

- **CQRS**는 **Command Query Responsibility Segregation**의 약자로,  
  "명령과 조회 책임 분리"라는 아키텍처 패턴입니다.
- 핵심 아이디어는  
  > 데이터를 **변경하는 작업(Command)**과 **조회하는 작업(Query)**을 서로 다른 모델 또는 저장소로 분리한다는 것.
- 보통 쓰기 작업(명령)과 읽기 작업(조회)을 분리해서 설계함으로써  
  성능 최적화, 확장성, 복잡도 관리에 도움을 줍니다.
- CQRS는 단일 데이터베이스뿐 아니라, 서로 다른 DB(예: 쓰기용은 RDBMS, 읽기용은 NoSQL)를 사용할 수도 있습니다.

---

## 3. CQRS vs MongoDB: 역할과 차이점

| 구분       | MongoDB                          | CQRS                           |
|------------|---------------------------------|--------------------------------|
| 종류       | 데이터베이스(DB) 시스템         | 아키텍처 설계 패턴             |
| 역할       | 데이터를 저장하고 조회하는 저장소 | 명령(Command)과 조회(Query) 책임 분리 설계 |
| 적용 대상  | 데이터 저장 방식과 관리          | 시스템 내 읽기/쓰기 요청 처리 구조 설계 |
| 특징       | NoSQL, 문서지향, 스키마 유연    | 읽기 모델과 쓰기 모델 분리 가능, 확장성 우수 |
| 사용 예시  | 애플리케이션의 데이터 저장소    | 대규모 시스템에서 복잡한 비즈니스 로직과 트래픽 분리 |

---

## 4. CQRS + MongoDB 조합

- MongoDB는 CQRS 아키텍처에서 **읽기 모델**이나 **쓰기 모델** 저장소로 사용할 수 있음
- 예를 들어:
  - 쓰기 작업은 RDBMS에서 처리하고,  
  - 읽기 작업은 MongoDB에서 처리해서 빠른 조회를 지원하는 경우도 많음.
- CQRS는 MongoDB에 국한되지 않고 다양한 DB 및 저장소와 함께 쓰일 수 있음

---

## 요약

- **MongoDB**: NoSQL 문서형 DB, 데이터 저장과 조회를 담당  
- **CQRS**: 읽기/쓰기 책임을 분리하는 아키텍처 패턴, 시스템 설계 원칙  
- 둘은 비교 대상이 아니라, 한쪽은 저장소(DB), 다른 한쪽은 설계방식(아키텍처)  
- MongoDB를 CQRS의 일부(읽기 또는 쓰기 저장소)로 활용 가능
